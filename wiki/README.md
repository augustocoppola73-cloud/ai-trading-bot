# AI Trading Bot LLM Wiki

This wiki is the persistent architectural memory for the AI Trading Bot project.
It is designed for human developers and future AI agents that need stable context
before changing the system.

The wiki is not a replacement for reading the code. It records the intent,
constraints, decisions, and concepts that should guide safe evolution.

## Project Shape

The current system is a Python trading research application built around:

- Streamlit for the interactive dashboard.
- Pandas DataFrames as the primary data model.
- Plotly for interactive portfolio and trade visualization.
- yfinance, Wikipedia, and CoinGecko as current market data and universe sources.

The main code lives in `src/`. Generated CSV artifacts live in `reports/`.

## How To Use This Wiki

Before significant changes:

1. Read the relevant architecture and constraint pages.
2. Inspect the current source code.
3. Propose changes before editing.
4. Update the wiki when architecture, decisions, concepts, or workflows change.

## Wiki Areas

- `architecture/`: module responsibilities, system flow, dashboard structure.
- `decisions/`: durable project decisions and tradeoffs.
- `concepts/`: trading and engine concepts used by the codebase.
- `metrics/`: performance and risk metric definitions.
- `strategies/`: strategy patterns and signal ideas.
- `visualization/`: dashboard and Plotly conventions.
- `experiments/`: future ideas, failed attempts, and research notes.
- `workflows/`: development, debugging, release, and AI collaboration process.
- `constraints/`: coding, performance, UI, and AI agent rules.
- `glossary/`: shared trading vocabulary.
- `prompts/`: reusable prompts for safe AI-assisted work.

## Governance

This wiki should evolve incrementally. Prefer small, precise updates after real
learning over broad autogenerated pages.
