# Development Workflow

For significant work:

1. Analyze the current code and data flow.
2. Summarize understanding.
3. Propose the implementation plan.
4. Identify impacted modules and risks.
5. Wait for explicit approval.
6. Make focused changes.
7. Run relevant verification.
8. Update this wiki when architecture or decisions change.

Avoid large refactors unless they have been discussed and approved.
